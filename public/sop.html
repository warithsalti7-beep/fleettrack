FleetTrack — Data Entry SOP

Standard Operating Procedure for dispatch / ops staff · Version 1.0 · April 2026
Audience: Dispatch office, admin assistants, fleet ops
Time commitment: 30–60 min setup, 15–30 min/week
Files to fill: 2 CSVs (fleet-setup + weekly-operations)
Upload frequency: Once for setup, weekly for operations
How this works: Every number on the FleetTrack dashboard — revenue, profit margin, VAT, driver scores, fleet utilization — is computed from the data in these two CSV files. Follow the SOP, every KPI updates automatically. Skip steps, dashboard shows stale numbers.

1. The two files

FileHow oftenWho fills itContent
fleet-setup.csvOnce, then rarelyOffice managerDrivers, vehicles, monthly fixed costs
weekly-operations.csvEvery MondayDispatch / opsShifts, trips, fuel, maintenance

Download template links are in the toolbar above (click "⬇" buttons). Both files use a record_type column — the system uses that to know what each row is. Leave any column blank if it doesn't apply.

2.4 Upload fleet-setup.csv

  1. Sign in as admin at https://fleettrack.no/login
  2. Sidebar → System → Settings → Data Import
  3. Paste your SEED_TOKEN (ask IT; stored in Vercel env)
  4. Drag fleet-setup.csv into the One-Shot Bundle Upload zone
  5. Review the preview → click Import to database
  6. Report shows e.g. DRIVER: 19 inserted, VEHICLE: 14 inserted, FIXED_COST: 13 inserted
You only do this once — or whenever a new driver/vehicle/cost is added.

4. Where to gather each data point

4.1 Bolt (trips, hours online, earnings)

  1. Go to fleets.bolt.eu → sign in as fleet manager
  2. Reports → Trips → select date range (e.g. 2025-09-01 to today) → Export CSV
  3. Column mapping to our TRIP rows:
    • Bolt Trip Date → date
    • Bolt Pickup Time → time
    • Bolt Driver → driver_email (match via internal directory)
    • Bolt Price → fare_nok
    • Bolt Distance → distance_km
    • Bolt Duration → duration_min
    • Bolt Rating → rating
    • platform = Bolt (hardcode)

4.2 Uber (trips per driver)

  1. Go to fleet.uber.com — fleet owner login
  2. Financials → Weekly Payments → each driver's statement
  3. Each driver can also export from the Uber Driver app: Earnings → Past Week → Export CSV
  4. Map to TRIP rows. platform = Uber

4.3 Norgestaxi / Oslo Taxi / 07000 (if you use a central)

  1. Log into the operator portal of whichever dispatch you work with
  2. Reports → Trip Log → date range → Export CSV
  3. platform = the central's name (e.g. Norgestaxi)

4.4 Circle K / Shell fleet card (fuel)

  1. Circle K Fleet portal: circlekfleet.com → Reports → Fleet Card Transactions → monthly CSV
  2. Shell: shellfleethub.com → similar flow
  3. Map each transaction to a FUEL row

4.5 Tesla (for Tesla vehicles in fleet)

  1. Tesla app → driver's account → Charging History
  2. Tesla sends CSV by email on request
  3. Or wait for Tesla Fleet API integration (roadmap)

4.6 Workshop / garage invoices

Most workshops email a PDF invoice. Create a Gmail rule: auto-forward to ops@fleettrack.no. Every Monday, dispatch enters the week's invoices as MAINTENANCE rows.

4.7 Driver-paid receipts (cash fuel, paper tickets)

Drivers photograph the receipt at end-of-shift → text to dispatch WhatsApp / email → dispatch enters into weekly file.

5. Upload & what happens next

  1. Sign in as admin at https://fleettrack.no/login
  2. Sidebar → System → Settings → Data Import
  3. Drag weekly-operations.csv into the One-Shot Bundle Upload zone
  4. Click Import to database
  5. Success report example:

8. Daily & weekly habits summary

  • Drivers: Just drive. At shift end, photograph any paper receipts and text to ops.
  • Dispatch daily (5 min): Photograph workshop invoices, note shift exceptions (no-shows, cancellations).
  • Dispatch Monday (15–30 min): Export Bolt + Uber + Norgestaxi CSVs for prior week → export fleet-card transactions → enter workshop invoices → combine everything into weekly-operations.csv → upload.
  • Office manager monthly: Review fixed costs in fleet-setup.csv. Update if insurance renewed, lease amount changed, new cost added.

9. Historical backfill (Sept 2024 → now)

To inject all your historical data in one go:

  1. Export trip data from each platform (Bolt, Uber, Norgestaxi) covering 2024-09-01 onwards — each as a CSV
  2. Export fuel card statements for the same period
  3. Gather workshop invoices for the same period
  4. Combine everything into one big weekly-operations.csv — every row tagged with record_type
  5. Upload once — system handles 50 000+ rows; expect ~2–5 min for huge backfills
  6. The dashboard then has 14+ months of history populating every chart, P&L, and driver scorecard
If a historical upload reports many errors, don't panic — fix the identified rows and re-upload the cleaned file. Duplicates from the successful rows will be auto-skipped.
