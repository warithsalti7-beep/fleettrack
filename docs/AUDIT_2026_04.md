# FleetTrack full audit — April 2026

_Scope: every public HTML page, every API route, Prisma schema, templates,
CI, and infrastructure. Flagged issues grouped by severity._

---

## TL;DR — the 7 things that matter most

1. **Auth is still client-side.** Passwords live in `public/auth.js`
   DEMO_USERS + `ft_custom_users` in localStorage. Server-side Next.js
   routes don't validate sessions — anyone who can hit `/api/drivers`
   gets the data. **Highest-impact remaining work.**
2. **Dashboard tables render at parse time** from baked-in arrays.
   `FleetData.refresh()` now exists but requires a page reload to see
   live DB data. Acceptable but not ideal.
3. **No rate limiting on any API route.** `/api/import/*` is gated by
   SEED_TOKEN; `/api/drivers` etc. are fully open. Must fix before prod.
4. **Audit log model exists but nothing writes to it yet.** Wire
   import success + permission change + role change into AuditLog inserts.
5. **Missing Norwegian P&L pieces resolved by FixedCost schema + template**
   but need to wire them into the dashboard P&L page. Data is in, rendering
   is not.
6. **Charts render but chart data is still hardcoded** in dashboard.html,
   not derived from FleetData. Visual regression if DB data differs
   significantly from demo numbers.
7. **Playwright tests cover 5 smoke scenarios.** Good baseline, but no
   tests for the import pipeline, no tests for role filtering, no tests
   for live-data banner.

## 1. Broken or half-finished features

### Critical
- **`/api/seed` only seeds demo data, not the user's imported CSVs.**
  Fine for dev, confusing in prod. Add a gate that refuses if FixedCost
  or Shift rows already exist.
- **`employee.html` exists but login now routes employees to `dashboard.html`.**
  `employee.html` is orphan — reachable only by typing `/employee` in the
  URL bar. Either delete it or redirect it.

### High
- **Export buttons on dashboard-level pages download client-side only.**
  They don't call `/api/export/*` (which exists and works against the DB).
  So if DB has fresher data than the table, exports diverge. Wire them
  to the API endpoints.
- **`Toast.info('… — not implemented yet')`** — 8 buttons still stubs
  (Add Driver, Add Vehicle, Log Service, Report Incident, etc.).
  Each needs a small form-modal and an API call.
- **"Refresh data" button doesn't exist on non-admin pages.** Drivers and
  employees can't force a live-data reload.

### Medium
- **Reports → PDF labels never materialised.** Every report downloads CSV.
  Fix is either: relabel to CSV uniformly (done earlier) OR add a real
  PDF renderer (server-side puppeteer, adds ~80 MB to cold start).
- **`/login.html` footer says v1.0.0** — hardcoded. Drive from
  `VERCEL_GIT_COMMIT_SHA`.

### Low
- Favicons on /driver and /employee pages look fine; /login could use
  a different theme-appropriate color.

## 3. Missing fleet KPIs / accounting

The FixedCost schema + template in commit `58d93f0` covers:
LEASE, INSURANCE, FINANCING, PARKING, WASH, LOYVE, TAXIMETER,
REGISTRATION, DEPRECIATION, OFFICE, SOFTWARE, SALARY, EMPLOYER_NI,
ACCOUNTING, OTHER.

### Still missing (add KPI cards for these when DB has data)
- **VAT payable** — accruing per trip at 12% (ride-share reduced rate).
  Need a Finance page that computes `sum(trip.fare) × 0.12` per quarter.
- **Input VAT reclaim** — 25% on fuel, maintenance, leases. Currently
  no rebate side accounted for.
- **Driver cost per km** — breakdown of payout / fuel / maintenance
  allocated to each trip km.
- **Revenue per km** — `fare / distance` — lets you spot trips that
  burn more in costs than they earn.
- **Dead-head km** — km driven without a rider. Need to log this via a
  "end-of-shift mileage" field per shift.
- **Customer acquisition** — N/A (Bolt/Uber controls demand) but a
  "tips per driver" KPI would be useful (missing entirely).
- **Break-even day count** — "we're break-even after the 7th of each
  month". Easy to derive once daily revenue × costs are live.

## 4. Role-based data isolation

### Verified ✓
- `driver.html` has `requireAuth(['driver'])` — admin/employee bounced.
- Dashboard nav filtering hides pages without matching permissions.
- Driver portal shows only `session.userId` stats, never fleet aggregates.

### GAPS (critical)
- **`/api/drivers` has NO auth.** A logged-out user can `curl` it and
  get the whole roster. Same for every other API route except
  `/api/seed` and `/api/import/*`.
- **`/api/export/*` routes have no auth** either — CSV exports of every
  table downloadable by anyone who knows the URL.
- **Driver with role='driver' hitting `/dashboard` URL** gets bounced by
  requireAuth — good — but a driver who promotes themselves to admin
  via localStorage surgery (`ft_role_drv-1 = 'admin'`) would bypass.
  This is the "fake auth" root problem.

### Medium
- `ft_preview_as` is not cleared on logout — an admin who previews-as,
  then logs out, then logs back in, starts in preview mode. Harmless but
  confusing.

## 5. Import pipeline soundness

### Critical — none
End-to-end trace:
- `templates/trips.csv` → `parseCsv()` in `src/lib/csv.ts` → validated
  in `/api/import/trips/route.ts` → Prisma insert with FK resolution.
- Error messages include row number + offending field. ✓
- Duplicate driver emails → rejected with actionable message. ✓

### High
- **No idempotency** on trip imports. Re-uploading the same `trips.csv`
  creates duplicate Trip rows. Add `unique(externalId)` or at least a
  "detect duplicates by (startedAt, driverId, vehicleId)" flag.
- **Transaction boundary is per-row.** If row 500 of 1000 fails, rows
  1-499 are committed. Need to decide: atomic bulk (all or nothing) or
  continue-on-error (current). Current is fine for long imports but
  document it.

### Medium
- **No CSV column validation up front.** A file missing `email` column
  silently skips every row. Add pre-check that required headers exist.
- **Date formats other than ISO** are accepted via JS `new Date()`.
  Locale-dependent. Restrict to `YYYY-MM-DD`.

- **"Data Import"** should appear in the sidebar at a single level, not

  admins can find it faster if promoted one level.
- **"Bonus"** nav group at the bottom — what is this? Looks like a
  scratchpad. Either promote to a real section or delete.

### Medium
- Several pages only reachable via URL hash (`#driver-coaching` etc.)
  with no sidebar link. Either add links or remove pages.
- Icon semantics: downward arrow `⬇` is used for both "open next
  section" and "download". Separate: use `▼` for nav, `⬇` for downloads.

## 7. Code health

### High
- **`dashboard.html` is 4 300 lines.** Only manageable because of strong
  section comments. At 5 000 it becomes unreviewable. Start extracting
  each `<div class="page">` into its own HTML file loaded on demand.
- **Duplicate data sources.** `auth.js DEMO_USERS` has 19 drivers,
  `fleet-data.js drivers[]` has 19 drivers. Unify — auth.js should
  derive from FleetData.
- **`driver.html driverStats`** — 19 hardcoded stat rows. Move to
  fleet-data.js (synthesise or merge from FleetData.drivers[].rev).

### Medium
- No `package.json` scripts for `lint` beyond eslint-next defaults.
- `@sentry/nextjs` not installed — we use the lightweight fetch reporter
  instead. Works but loses session replay server-side. Decide later if
  worth it.
- `.env.example` lists 3 keys; should list all optional ones from
  `src/lib/env.ts` (SENTRY_DSN, ANTHROPIC_API_KEY, SEED_TOKEN, etc.).

### Low
- `TODO` / `FIXME` grep: 0 hits. Clean.
